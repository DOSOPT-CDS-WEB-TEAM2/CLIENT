<h2>  📄 컨벤션 및 브랜치 전략 </h2>
<h3>커밋 컨벤션<h3>

- init: 개발 환경 초기 세팅
- feat: 새로운 기능 추가 (새로운 구현)
- fix: 버그 수정
- docs: 문서 추가, 수정, 삭제
- style: 코드 포맷팅(코드 순서 변경, 세미콜론 추가)
- design: CSS 등 사용자 UI 디자인 변경
- refactor: 코드 리팩토링
- chore: 그 외 자잘한 수정

<br/>


<h3>브랜치 컨벤션<h3>

`페이지명/#이슈번호`

<br/>

<h3>브랜치 운영<h3>

- **main**: 최종 merge하는곳
- **feature/페이지명/#이슈번호**: 각 페이지별 기능을 개발할 때 사용한 브랜치
